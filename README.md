# Hang-Compiler
A C-like language compiler implemented with Flex &amp; Bison, supporting AST construction, IR (Quaternary) generation, and x86 assembly backend.
使用 Flex 和 Bison 实现的类 C 语言编译器，支持 AST 构造、IR（四元）生成和 x86 汇编后端。

## 核心特性
- **词法分析**：使用 Flex 处理字符集，支持整数、标识符、运算符及单行/多行注释。
- **语法分析**：基于 Bison 构建 LALR 语法分析器，支持嵌套作用域、If-Else、While 循环等标准文法。
- **语义分析**：通过**符号表**管理标识符属性，实现变量声明检查及类型匹配。
- **中间代码**：生成**四元式 (Quaternary)**，作为前端与后端解耦的中间表示。
- **目标代码**：实现了四元式向 **x86 汇编**的翻译映射，并支持通过 **NASM** 编译为可执行文件。

## 编译流程说明
该编译器遵循经典的五阶段模型：
1. **词法分析** -> 产生 Token 流。
2. **语法分析** -> 构建 **抽象语法树 (AST)**。
3. **语义分析** -> 填充符号表并检查逻辑。
4. **中间代码生成** -> 产生四元式序列。
5. **代码生成** -> 翻译为 `.asm` 汇编文件。

## 运行环境
- Flex / Bison
- GCC (C99 标准)
- NASM (用于汇编后端代码)

# FANG-Compiler (hcc)

这是一个基于 **Flex** & **Bison** 开发的类 C 语言（名为 FANG 语言）自底向上编译器。本项目实现了从高级语言源码到 **x86 汇编代码**的完整编译流程，并支持通过自动化脚本生成可执行二进制文件。

## 🚀 核心特性

- **前端解析**：基于 Flex 和 Bison 构建 LALR 语法分析器，支持嵌套作用域、While 循环、If-Else 分支等。
- **抽象语法树 (AST)**：自底向上构建语法树，清晰展示程序逻辑结构。
- **语义分析与符号表**：建立符号表作为“唯一权威数据源”，负责变量声明检查与类型匹配。
- **中间表示 (IR)**：生成**四元式 (Quaternary)** 序列，实现前后端逻辑解耦。
- **后端代码生成**：将四元式直接映射为 **x86 汇编 (NASM 格式)**，支持 Linux 64 位环境运行。

## 📂 项目结构与文件说明

为了方便查阅，本项目保留了开发过程中的全部文件（包含工具生成的中间件）：

| 文件/目录 | 说明 |
| :--- | :--- |
| **src/** | 存放编译器核心源码（.l, .y, .c, .h） |
| **src/lexer.l** | 词法分析定义（Flex） |
| **src/parser.y** | 语法规则与 AST 构建定义（Bison） |
| **src/semantic.c** | 语义分析与符号表逻辑 |
| **src/ir_gen.c** | 四元式生成逻辑 |
| **src/codegen.c** | x86 目标代码生成映射 |
| **build.sh** | **自动化构建脚本**：一键完成编译、汇编、链接 |
| **Makefile** | 编译配置文件，定义了 Flex/Bison 的转换规则 |
| **test.hang** | 示例 FANG 语言源文件，用于功能测试 |
| **[Generated]** | 仓库中保留了 `lex.yy.c` 等生成件，方便无环境用户查阅中间逻辑 |

## 🛠️ 使用方法 (Quick Start)

本项目已集成自动化脚本，仅需两步即可运行。

### 1. 环境准备
确保您的 Linux 环境已安装：`flex`, `bison`, `gcc`, `nasm`, `make`。

### 2. 一键编译与运行
使用提供的 `build.sh` 脚本，传入源文件即可自动完成：**清理 -> 构建编译器 -> 翻译源码 -> 汇编 -> 链接**。

```bash
# 1. 赋予执行权限
chmod +x build.sh

# 2. 运行自动化构建 (以 test.hang 为例)
./build.sh test.hang

# 3. 运行最终生成的二进制程序
./test
