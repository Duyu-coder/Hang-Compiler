# HANG-Compiler (hcc)

这是一个基于 **Flex** & **Bison** 开发的类 C 语言（名为 HANG 语言）的自底向上编译器。本项目实现了从高级语言源码到 **x86 汇编代码**的完整编译流程，并支持通过自动化脚本生成可执行二进制文件。

## 🚀 核心特性

- **前端解析**：基于 Flex 和 Bison 构建 LALR 语法分析器，支持嵌套作用域、While 循环、If-Else 分支等。
- **抽象语法树 (AST)**：自底向上构建语法树，清晰展示程序逻辑结构。
- **语义分析与符号表**：建立符号表作为“唯一权威数据源”，负责变量声明检查与类型匹配。
- **中间表示 (IR)**：生成**四元式 (Quaternary)** 序列，实现前后端逻辑解耦。
- **后端代码生成**：将四元式直接映射为 **x86 汇编 (NASM 格式)**，支持 Linux 64 位环境运行。

## 🧠 核心原理与架构设计

本项目遵循经典的编译器五阶段设计。为了理清符号表管理与 x86 寄存器映射的复杂逻辑，在开发初期绘制了详细的编译流程图：

**技术关键点：**
1. **符号表枢纽**：符号表记录了所有变量的作用域、类型及内存偏移量，是语法分析与代码生成阶段的数据核心。
2. **自动化链路**：后端生成纯正的 x86_64 汇编，通过 `nasm` 汇编后，利用 `ld` 直接进行系统调用级链接，不依赖标准 C 库 (CRT)，体现了对操作系统底层接口的直接操作。

## 🛠️ 运行环境与使用方法

### 环境要求
- **Flex / Bison**: 词法与语法分析工具
- **GCC**: 用于编译编译器主体
- **NASM / LD**: 用于后端汇编与链接 (Linux x86_64)

### 快速开始
本项目已集成自动化脚本 `build.sh`。只需传入源文件，脚本将自动完成：**清理 -> 构建编译器 -> 翻译源码 -> 汇编 -> 链接**。

```bash
# 1. 赋予执行权限
chmod +x build.sh

# 2. 运行自动化构建 (以 test.hang 为例)
./build.sh test.hang

# 3. 运行最终生成的二进制程序
./test
